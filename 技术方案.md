本项目包含五个子系统，覆盖了从内容采集、处理、存储到全自动推送的完整数据流水线。

## 1. 新闻定时推送系统

### 架构

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────┐
│ Reuters RSS  │     │Bloomberg RSS │     │ translate.py │     │  hermes  │
│ (Google News │     │ (feeds.bloom │     │ (LLM 翻译    │     │  send    │
│   RSS 中转)  │     │  berg.com)   │     │  中英对照)   │     │  → QQ)   │
└──────┬───────┘     └──────┬───────┘     └──────┬───────┘     └────┬─────┘
       │                    │                    │                    │
       ▼                    ▼                    ▼                    ▼
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────┐
│ reuters_push │     │ bloomberg_   │     │ reuters_     │     │ 本地     │
│     .py      │     │ fetch_and_   │     │ latest.md   │     │ Markdown │
│              │     │ notify.py    │     │ bloomberg_  │     │  文件    │
└──────────────┘     └──────────────┘     │ latest.md   │     └──────────┘
                                            └──────────────┘
```

### 核心模块

| 文件 | 职责 |
|---|---|
| `reuters_fetcher.py` | Google News RSS 抓取 Reuters 新闻，提取 title/url/summary/pub_date |
| `reuters_push.py` | 抓取 → 翻译 → 写 Markdown → `hermes send` 推送 |
| `bloomberg_rss_fetch.py` | Bloomberg Markets RSS 抓取 + 存储 ChromaDB |
| `bloomberg_fetch_and_notify.py` | 抓取 → 翻译 → 写 Markdown → `hermes send` 推送 |
| `translate.py` | 批量调用 LLM API（OpenAI 兼容）翻译标题+摘要为中文 |
| `news_sources.py` | 来源元数据 enrichment（domain/transport 标准化） |

### 数据流

1. **抓取** → RSS/HTTP 拉取原始新闻列表（去重 + 排序）
2. **翻译** → 批量发送至 LLM API，中英对照输出
3. **格式化** → 生成 Markdown 文件（标题、来源、总结、发布时间）
4. **推送** → `hermes send -t qqbot -f <markdown>` 发送到 QQ

### 调度

| 任务 | 北京时间 | Cron |
|---|---|---|
| Reuters 早报 | 09:00 | `0 1 * * *` UTC |
| Reuters 晚报 | 21:00 | `0 13 * * *` UTC |
| Bloomberg 早报 | 09:00 | `0 1 * * *` UTC |
| Bloomberg 晚报 | 21:00 | `0 13 * * *` UTC |

### 环境依赖

| 变量 | 说明 |
|---|---|
| `HERMES_NEWS_TARGET` | 推送目标（`qqbot`） |
| `TRANSLATE_LLM_BASE_URL` | LLM API 地址 |
| `TRANSLATE_LLM_API_KEY` | LLM API 密钥 |
| `TRANSLATE_LLM_MODEL` | 翻译模型（默认 `z-ai/glm-5.1`） |

## 2. 文档处理流水线（Hermes Document Pipeline）

### 架构

```
┌─────────────────┐
│   输入文件       │
│  (PDF/DOCX/      │
│   XLSX/TXT…)    │
└────────┬────────┘
         │
         ▼
┌──────────────────────────────────────────┐
│           auto_router.py                  │
│  关键词识别 → 发票/文档 自动路由           │
└──────────────┬───────────────────────────┘
               │                    │
       ┌───────▼──────┐   ┌────────▼────────┐
       │ FinanceEngine │   │ DocumentEngine  │
       │  (发票处理)   │   │  (文档处理)      │
       └───────┬──────┘   └────────┬────────┘
               │                    │
               ▼                    ▼
       ┌──────────────┐   ┌─────────────────┐
       │  UPSERT D1   │   │  文本提取        │
       │  invoices表  │   │  (pdf_inspector  │
       └──────┬───────┘   │   + docx/openpyxl│
              │           └────────┬─────────┘
              ▼                    │
       ┌──────────────┐            ▼
       │ sync_to_     │   ┌─────────────────┐
       │  ChromaDB    │   │  归档拷贝        │
       │  invoices集  │   │  (MD5去重)       │
       └──────────────┘   └────────┬─────────┘
                                    │
                                    ▼
                           ┌──────────────┐
                           │  UPSERT D1   │
                           │  documents表 │
                           └──────┬───────┘
                                  │
                                  ▼
                           ┌──────────────┐
                           │ sync_to_     │
                           │  ChromaDB    │
                           │  documents集 │
                           └──────────────┘
```

### 核心模块

| 文件 | 类 | 职责 |
|---|---|---|
| `hermes_core.py` | — | 配置加载（`.env` → 环境变量）、D1 HTTP Session（自动重试） |
| `core_engine.py` | `HermesProjectCore` | 基类：`query_d1` / `sync_to_chroma` / `archive_file` / `get_md5` |
| `invoice_engine.py` | `FinanceEngine` | 发票 UPSERT → D1 + ChromaDB，按购买方报表 |
| `doc_engine.py` | `DocumentEngine` | 多格式文本提取 → 归档 + D1 + ChromaDB |
| `auto_router.py` | `AutoRouter` | 关键词检测 → 自动路由到发票/文档引擎 |
| `pdf_engine.py` | — | PDF 文本提取（调用 `pdf_inspector`） |

### D1 表结构

```sql
-- 发票表
CREATE TABLE invoices (
    id             INTEGER PRIMARY KEY AUTOINCREMENT,
    invoice_number TEXT UNIQUE NOT NULL,
    invoice_date   TEXT,
    buyer_name     TEXT,
    seller_name    TEXT,
    item_name      TEXT,
    amount_net     REAL,
    tax_amount     REAL,
    total_amount   REAL,
    file_path      TEXT,
    created_at     TEXT DEFAULT (datetime('now'))
);

-- 文档表
CREATE TABLE documents (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    title      TEXT,
    company    TEXT,
    category   TEXT,
    summary    TEXT,
    tags       TEXT,
    file_path  TEXT UNIQUE NOT NULL,
    created_at TEXT DEFAULT (datetime('now'))
);
```

### 支持的文件格式

| 格式 | 提取方式 |
|---|---|
| PDF | `pdf_inspector`（本地提取，无需 OCR） |
| DOCX | `python-docx`（段落 + 表格） |
| TXT/MD/LOG/CSV | 多编码探测（utf-8 → gbk → latin-1） |
| XLSX/XLS | `openpyxl` → Markdown 表格 |

### 归档命名规则

```
YYYYMMDD_<公司前10字>_<标题前15字>_<MD5前8位>.<ext>
```

### 配置加载顺序

1. `$HERMES_PROJECT_ENV`（显式指定路径）
2. `<project_root>/.env`
3. `$HERMES_HOME/.env`
4. 进程环境变量
5. 内置默认值

## 3. 纷传圈子监控系统

### 架构

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────┐
│ Token 验证   │───▶│ HTTP 轮询    │───▶│ HTML 解析    │───▶│ SQLite   │
│ (last_token) │    │ (圈子 API)   │    │ (文章提取)   │    │ 去重     │
└──────────────┘    └──────────────┘    └──────┬───────┘    └────┬─────┘
                                                  │                 │
                                                  ▼                 ▼
                                           ┌──────────────┐    ┌──────────┐
                                           │ ChromaDB     │    │ 有新内容  │
                                           │ sx_essentials │    │ → 推送    │
                                           │ 语义索引     │    │ → QQ     │
                                           └──────────────┘    └──────────┘
```

### 核心流程

1. **Token 管理**：从 `last_token.txt` 读取/刷新会话 Token
2. **HTTP 轮询**：调用圈子 API 拉取最新文章
3. **HTML 解析**：提取文章标题、内容、时间、作者
4. **SQLite 去重**：MD5 hash 比对，跳过已处理文章
5. **ChromaDB 索引**：语义向量入库，支持相似度检索
6. **条件推送**：仅当发现新内容时通知用户

### 调度策略

| 时段 | 频率 | Cron |
|---|---|---|
| 12:00-19:00 | 每 2 小时 | `0 4,6,8,10 * * *` UTC |
| 19:00-24:00 | 每 30 分钟 | `0,30 11-15 * * *` UTC |

### 数据存储

| 存储 | 路径 | 用途 |
|---|---|---|
| SQLite | `/root/sx_articles.db` | 文章去重 + 持久化 |
| ChromaDB | `/root/.hermes/chroma` / `sx_essentials` | 语义索引 |
| Token | `/root/last_token.txt` | 会话认证 |

## 4. 腾讯会议自动机器人

### 架构

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ 消息解析     │───▶│ 调度器       │───▶│ 浏览器 RPA   │
│ (会议号/密码  │    │ (cron 每分钟  │    │ (Playwright  │
│  /时间提取)   │    │  扫描 schedule)│   │  + Chromium) │
└──────────────┘    └──────────────┘    └──────┬───────┘
                                                │
                                                ▼
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│ ChromaDB     │◀───│ AI 摘要      │◀───│ Whisper      │
│ 会议存储     │    │ (全文总结)    │    │ 语音转写     │
└──────────────┘    └──────────────┘    └──────┬───────┘
                                                │
                                                ▼
                                         ┌──────────────┐
                                         │ ffmpeg 录音   │
                                         │ (PulseAudio   │
                                         │  虚拟声卡)    │
                                         └──────────────┘
```

### 媒体层

| 组件 | 用途 |
|---|---|
| Xvfb `:99` | 虚拟显示（无头服务器运行浏览器） |
| PulseAudio | 虚拟声卡（`virtual_meeting` sink） |
| ffmpeg | 从 PulseAudio monitor 录制音频（16kHz 单声道 PCM） |
| Playwright | 控制 Chromium 浏览器加入腾讯会议 |

### 核心流程

1. ** NLP 解析**：从自然语言消息提取会议号/密码/时间
2. **语音录制**：每 3 分钟切割一个 WAV 文件
3. **实时转写**：Whisper 逐段转写
4. **AI 摘要**：每段生成摘要，结束后合并全文总结
5. **入库**：ChromaDB + D1 存储会议内容
6. **推送**：通知 Telegram
